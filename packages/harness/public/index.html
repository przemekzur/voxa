Service connectors

Each connector is a small plug-in that turns a service into voice tools for the Voxa cockpit. Enable one, configure it, and its actions appear on the bridge at this server.

How connectors work ▾
  1. Discover — every folder in connectors/<id>/index.mjs is auto-loaded as a connector.
  2. Enable & configure — flip a connector on and fill its fields (API keys, hosts). Secrets stay server-side.
  3. Test — run its connectivity check and try each action right here before going live.
  4. Speak — enabled actions are merged into GET /api/voice/tools; the cockpit calls them by voice.

Want a new one? A connector is a single dependency-free ES module — see BUILDING-A-CONNECTOR.md, or ask Claude to run the connector-builder skill to scaffold it for you.
